Detailed Description

the simulator that runs the simulation on the given input

This is responsible for running steps of the algorithm until it should halt. It takes a internal state which provides the core implementation of the algorithm and allows seamless changing between implementations.

Updates are sent to the aggregator. For performance reasons the simulation state is mutated rather than new ones created. This means that it is up to the aggregator to copy the data if it needs to retain it.
